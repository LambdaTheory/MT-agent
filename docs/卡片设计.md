# MT-agent 飞书卡片重构设计文档

本文档用于记录和同步 MT-agent 中飞书卡片的二次设计评审结果及重构进度。各模块的 Session 可以在此文档中记录相关卡片的修改进度、踩坑记录或状态同步。

## 1. 设计基线与核心目标 (基于 `lark-card-designer` 规范)

- **决策前置，详情后置**：首屏第一句话必须是结论、风险提示或待决策动作。
- **结构化与密度控制**：坚决摒弃将完整 JSON 参数直接放在首屏。复杂长列表必须折叠或限制展示数量（如 Top 3/5）。
- **统一视觉与语义规范**：
  - 🔴 **红色 (Red)**：阻断、高风险、不可回滚、严重错误。
  - 🟠 **橙色 (Orange)**：需要注意、待确认、人工介入、告警。
  - 🟢 **绿色 (Green)**：成功、健康、完成。
  - 🔵 **蓝色 (Blue)**：信息、处理中、常规状态。
- **固定审计与元数据栏**：卡片底部必须保留 `runId`、`decisionId`、`requestRef` 等审计追踪数据。

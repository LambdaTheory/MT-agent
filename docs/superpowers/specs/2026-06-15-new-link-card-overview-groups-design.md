# 新链接卡片概览与品类分组设计

## 背景

当前飞书卡片的“新链接冷启动”模块偏重状态解释和 Top10 明细。实际查看时，用户需要先看到新链总体规模和健康情况，再展开查看全部端内 ID，并按商品种类快速定位，例如 `大疆 Pocket 3：721，722，723`。

本设计只调整飞书卡片展示方式，不改变新链接池筛选、商品总表 first-seen 状态、Excel、Markdown 或原始 JSON 的业务含义。

## 目标

- 在“新链接冷启动”模块顶部展示新链总体概览。
- 使用飞书 `column_set` 分栏展示 `总数`、`健康`、`亚健康`。
- 在折叠明细中展示所有新链接端内 ID。
- 明细按商品种类/短名聚合，格式为 `大疆 Pocket 3：721，722，723`。
- 保持越新的链接越优先展示。

## 非目标

- 不新增抓取来源。
- 不改变新链接池是否入池的判定。
- 不改变 Excel、Markdown、报告上下文结构。
- 不实现交互审批或 LLM 能力。

## 健康口径

新链接仍沿用现有冷启动状态：

- 健康：`强跑通`、`优秀链接`、`访问达标`
- 亚健康：`有苗头`、`未启动`、`危险`、`待观察`

没有匹配到报表行的链接归为 `待观察`，因此计入亚健康。

## 卡片结构

标题保持为 `新链接冷启动（N）`。

模块内容分为两层：

1. 概览分栏
   - `总数`：全部新链接数量。
   - `健康`：健康状态链接数量。
   - `亚健康`：亚健康状态链接数量。

2. 折叠明细
   - `健康链接`：按商品种类聚合端内 ID。
   - `亚健康链接`：按商品种类聚合端内 ID。

折叠明细必须覆盖所有新链接端内 ID，不做 Top10 截断。

## 商品种类映射

商品种类名复用现有商品短名能力，保证与其他飞书表格显示一致。

映射优先级：

1. 使用现有 `product-name-map.json` 中的手工短名映射。
2. 使用现有商品名清洗和型号识别规则，例如 `大疆 Pocket 3`、`iPhone 15 Pro Max`。
3. 商品名仍为空时使用 `未识别品类`。

实现上应避免复制一份新的正则规则。现有 `productDisplayName.ts` 需要暴露或复用一个适用于新链接池 item 的短名解析入口。

## 排序规则

- 新链接行按 `submittedAt` 倒序。
- 商品种类分组按组内最新 `submittedAt` 倒序。
- 同一商品种类内 ID 按 `submittedAt` 倒序。
- 同一时间的 ID 使用数字 ID 升序作为稳定 tie-breaker。

## 数据流

输入继续使用 `context.newProductPoolItems`。

卡片构建阶段新增一个纯 view-model 层，将新链接池转换为：

- `totalCount`
- `healthyCount`
- `subHealthyCount`
- `healthyGroups`
- `subHealthyGroups`

飞书卡片只消费该 view model，避免统计、分组、格式化逻辑散落在组件构造代码中。

## 空态和异常

- 无新链接时沿用现有模块显示行为，不扩大范围。
- 商品名无法解析时进入 `未识别品类`。
- 日期无效时排序优先级最低。
- 无报表行匹配的新链接仍展示在亚健康明细中。

## 测试要求

- 概览分栏展示 `总数`、`健康`、`亚健康`。
- 健康/亚健康计数符合确认口径。
- 折叠明细包含全部新链接 ID，不只 Top10。
- 商品种类聚合输出类似 `大疆 Pocket 3：721，722，723`。
- 分组和组内 ID 按最新提交时间优先排序。
- 无报表行匹配的链接进入亚健康。

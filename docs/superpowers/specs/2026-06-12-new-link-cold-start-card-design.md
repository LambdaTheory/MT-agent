# 新链接冷启动卡片设计

## 背景

goods-manager 新品池当前实际代表“最近提交的新链接”，不是新品类或新实物商品。同一个品可能存在多条链接，因此日报应考核链接冷启动效果，而不是判断新品孵化是否成功。

现有飞书卡片只展示 `新品维护池（N）` 和前 10 个商品摘要，无法回答整体冷启动健康度、链接卡在哪个阶段、今天优先处理哪些链接。

## 目标

- 将卡片展示文案从“新品维护池”调整为“新链接冷启动”。
- 以最近 7 天提交的 goods-manager 链接作为考核池入口。
- 按链接上线时间精确折算日均商品页访问。
- 在飞书卡片中展示整体分层、认可线、优秀线和优先处理明细。
- 保持 Excel 明细和上下文结构不变，避免影响现有日报产物。

## 口径

- 考核对象：goods-manager 最近 7 天提交的商品链接。
- 上线时间：`newProductPoolItems[].submittedAt`。
- 上线时长小时：报告日期当日 `23:59:59` 减去上线时间。
- 有效上线天数：`max(上线时长小时 / 24, 0.25)`，最少按 6 小时折算，避免刚上线链接被放大。
- 日均访问：该链接 `7d.dashboardVisits / 有效上线天数`。
- 访问指标：使用日报中的“商品页访问”。
- 成交指标：使用日报中的 `7d.shippedOrders`。

## 分层标准

- 强跑通：`7d.shippedOrders >= 1`。
- 优秀链接：无成交时，`日均访问 >= 10`。
- 访问达标：无成交且非优秀时，`日均访问 >= 6`。
- 有苗头：无成交且 `日均访问 >= 3`。
- 未启动：无成交且 `日均访问 < 3`。
- 危险链接：无成交且上线满 72 小时后 `7d.dashboardVisits = 0`，或 `日均访问 < 1`。

危险链接是行动优先级标签；统计上归入“危险”，不再重复计入“未启动”。

## 卡片展示

飞书卡片新增一个“新链接冷启动”折叠面板，替换原 `新品维护池` 面板。

面板顶部展示整体摘要：

- `近7天链接 N 条`。
- `强跑通 A｜访问达标 B｜有苗头 C｜未启动 D｜危险 E`。
- `平均访问 X/天｜认可线 >=6/天｜优秀线 >=10/天`。

面板内展示 Markdown 化的分层和明细，避免飞书 `collapsible_panel` 内不支持 `table` 组件导致卡片发送失败：

1. 分层状态：状态、条数、判断标准、今日动作。
2. 优先处理链接：链接、上线、日均访问、访问、成交、状态。

明细最多展示 10 条，排序优先级为：危险、未启动、有苗头、访问达标、优秀链接、强跑通；同状态下日均访问低的排前面。

## 兼容与降级

- 如果只有 `newProductPoolIds`，没有 enriched item，则展示“新链接冷启动（N）”和 ID 待观察列表。
- 如果链接无法匹配日报商品行，则访问和成交显示为 0，状态显示为“待观察”。
- 如果链接提交时间晚于报告数据日期，则状态显示为“待观察”，避免用尚未覆盖该链接的访问数据误判为危险。
- 如果 `submittedAt` 无法解析，则上线显示为 `-`，日均访问按 7 天口径保守折算。
- 未配置 goods-manager 时，不显示新链接冷启动面板。

## 不做

- 不修改 goods-manager API。
- 不新增数据库或持久化状态。
- 不改变 Excel `新品池维护` sheet。
- 不改变日报抓取、汇总和分析规则。

## 测试

- 覆盖强跑通、访问达标、有苗头、未启动、危险和待观察状态。
- 覆盖卡片文案从“新品维护池”调整为“新链接冷启动”。
- 覆盖 fallback `newProductPoolIds` 展示。
